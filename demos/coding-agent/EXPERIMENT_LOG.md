# Experiment log: coding-agent

Started 2026-06-08. See demos/coding-agent/plan.md.

- 2026-06-08: Built the coding-agent task-graph scenario (4th viz tab). Task: "summarize a paper that just came out, then implement it" — chosen because a post-cutoff paper is the purest not-one-shottable-without-search case. Ran it for real: WebSearch (×2) → WebFetch (×4, arXiv abstract + full HTML + GitHub repo + raw reference code) surfaced **p-less sampling** (arXiv:2509.23234, Sept 2025), a hyperparameter-free truncation sampler. Extracted the algorithm (the plan): threshold = collision likelihood L[P]=ΣP(v)² (=exp(-H₂), Rényi-2 entropy), keep tokens with P(v)≥L[P], renormalize, sample; argmax always survives since maxP≥ΣP² so the kept set is non-empty. Implemented it pure-stdlib/CPU in `demos/coding-agent/generated/p_less.py` (softmax, collision_likelihood, p_less_filter, p_less_sample) and wrote `tests/unit/test_p_less.py` (9 cases incl. uniform-keeps-all, peaked-keeps-top, argmax-always-survives, flatter-keeps-more, seeded determinism) → all green (the real verify node). New graph builder `build_coding_agent_task_graph()` produces the search→plan→codegen→verify diamond with typed edges (informs / plans / verifies); `CodingNode`/`CodingEdge`/`CodingAgentTaskGraph` added to modules/proof_server/task_graph.py; 5 builder tests added. Captured the real trace into `coding_agent_graph.json` (10 nodes: 2 search + 4 fetch + 1 plan + 2 codegen + 1 verify; 10 edges) and rendered it as a 4th tab (column layout: retrievals → plan → codegen → verify) on https://taskgraph-dss.surge.sh. demo.sh prints the graph and re-runs the verify node (the retrieval/plan/codegen nodes are a captured one-time trace; verify is genuinely re-runnable). Full unit suite 563 passed / 2 skipped.
