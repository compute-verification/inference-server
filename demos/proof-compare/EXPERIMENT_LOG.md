- 2026-06-09: Coding-agent abstraction fix + frontend rewrite. (1) The coding tracer's nodes were agent ACTIONS (prompt/search/fetch/plan/codegen/test) — a single "plan" node folded a 2900-token generation into one fake forward pass, mis-counting FLOPs and breaking the "one node = one forward pass" invariant every other scenario honours. Rewrote `tracers/coding.py` at forward-pass granularity: each LLM turn = one `prefill` over the tokens newly added to context (the prompt, or a tool's returned output) + one `decode` per generated token; tool calls run no forward pass so they're NOT nodes — their output becomes the next prefill (tagged `payload.via`). The run is now a single prefill/decode chain (68 nodes for the demo run). Prefill `attended` is the causal triangle *difference* `(C+p)(C+p+1)/2 − C(C+1)/2` (reduces to `p(p+1)/2` at C=0, matching inference). Dropped the six coding-only kinds from `graph.py` KINDS (nothing dispatches on kind). (2) Replaced the hand-rolled single-file SVG renderer with a Vite + React + React Flow (@xyflow/react v12) + elkjs app under `viz/` (user picked this stack after a build-vs-no-build options review). `build_all.py` no longer bakes `const DATA` into HTML — it writes `graphs.json` to `traces/` and `viz/public/`, which the app fetches. Custom `TaskNode` keeps the card UI (kind, label, exact FLOPs, proportional bar, colour by kind); edges keep rejected/fan-in/handoff styling. Adversarial review (general-purpose agent) found no BLOCKER/MAJOR; applied its 2 cheap MINORs (fetch abort guard, edgeStyle null guard). Roadblock: a 68-node chain under `fitView` collapsed to an unreadable ribbon → enabled elk `MULTI_EDGE` wrapping at aspectRatio 1.7 + capped `maxZoom` 1.4. Verified headlessly with playwright/chromium against the LIVE surge URL — all four tabs render with correct node/edge counts + FLOPs totals, zero console errors. dist/ + node_modules/ gitignored.
