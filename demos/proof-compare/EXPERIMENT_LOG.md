# Experiment log: proof-compare

Started 2026-06-04. See demos/proof-compare/plan.md.

- 2026-06-04: Initial build — new "proof server" (compare + task-graph variant) downstream of the tap-protocol topology, in worktree `proof-compare-graph`. Added `modules/proof_server/task_graph.py` (`Task`/`TaskGraph` dataclasses, `forward_flops()` option-2 estimate `2·n_params·tokens + 4·n_layers·d_model·ctx·tokens`, `build_task_graph()` emitting one fat prefill task + a thin decode chain linked by `next`). Two deliberate approximations, both because the real inputs aren't available in the mock demo: whitespace tokenization (mock outputs are canned `MOCK[...]` strings, so real BPE is meaningless) and a static `MODEL_DIMS` lookup keyed by manifest `model.source` (the manifest pins `config.json` by digest + `hf://` source rather than embedding `n_params/n_layers/d_model`). Added `demos/proof-compare/servers/proof_server.py` (`POST /compare` bitwise-compares host vs recomp output, logs MATCH/MISMATCH, builds+writes the graph, never lets a graph-build failure break the compare path; `GET /health` exposes counters). Data path chosen (per design discussion): Tap forwards both outputs rather than each cluster fanning out independently — smaller diff since recomp already holds the host output. Patched `recomp_cluster.py` `/verify` to also return `recomp_output`, and `tap.py` with `--compare-server-url` + `_async_compare` (both additive, so the existing `demos/proof-server` SP1 demo is unaffected). New `demo.sh` is a trimmed clone of the proof-server demo (host+recomp+tap+gateway+proof server, all `--mock`, no SP1/GPU). Verified: `tests/unit/test_task_graph.py` (14 cases) green; full unit suite 524 passed / 2 skipped; `demo.sh → ALL PASS` (2 compared, 2 matches, 2 graphs built; prefill 6.88 GFLOP vs decode 3.44 GFLOP on a 2-token prompt).
