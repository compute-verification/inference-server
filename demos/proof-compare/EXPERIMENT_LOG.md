- 2026-06-09: Collapsible long chains (data stays atomic). The realistic coding graph is 6405 forward-pass nodes — honest but a dense block to read. Added a view-layer collapse so long linear runs fold into one group node, keeping graphs.json fully atomic (one node = one forward pass). New `viz/src/collapse.js` (pure): `segmentGraph` partitions the graph into maximal 1:1 same-(kind,phase) runs (fan-in/fan-out break runs into singletons); `buildDisplayGraph(expandedSet)` emits a group node per collapsed run (summed FLOPs/tokens/count) and remaps edges to display ids (drop internal/self edges, dedup). Grouped by **kind+phase** (not phase alone) so a turn's prefill "jump" stays a separate orange node and only its decode stream collapses → coding shows 12 nodes (5 prefills + 7 decode groups) by default; click a group to expand to atomic, click an expanded node / Collapse-all to fold back; auto-fit on toggle; meta always reports atomic totals (6405 · 1.41 PFLOP). Inference/spec/training have no long runs → untouched. Tracer change: `coding.py` tags each node `payload.phase` (the turn label) so groups read "write p_less.py" not "codegen" — still atomic, just a richer label. Tooling: added **vitest** (first JS tests in the repo) — 13 cases covering segmentation thresholds, phase/fan-in/fan-out splits, edge remap (no self-loops/dangles/dupes), expand↔collapse round-trip + idempotence. Adversarial review confirmed atomicity + correctness, no blocker/major; caught one real artifact — the kind+phase separator had become a NUL byte (made git see collapse.js as binary) → switched to `JSON.stringify([kind, phase])` (collision-proof, clean text). Verified headlessly on the LIVE site: 12↔6405 toggles, single-group expand, other tabs clean, zero console errors. Live: https://taskgraph-dss.surge.sh
- 2026-06-09: Coding-agent abstraction fix + frontend rewrite. (1) The coding tracer's nodes were agent ACTIONS (prompt/search/fetch/plan/codegen/test) — a single "plan" node folded a 2900-token generation into one fake forward pass, mis-counting FLOPs and breaking the "one node = one forward pass" invariant every other scenario honours. Rewrote `tracers/coding.py` at forward-pass granularity: each LLM turn = one `prefill` over the tokens newly added to context (the prompt, or a tool's returned output) + one `decode` per generated token; tool calls run no forward pass so they're NOT nodes — their output becomes the next prefill (tagged `payload.via`). The run is now a single prefill/decode chain (68 nodes for the demo run). Prefill `attended` is the causal triangle *difference* `(C+p)(C+p+1)/2 − C(C+1)/2` (reduces to `p(p+1)/2` at C=0, matching inference). Dropped the six coding-only kinds from `graph.py` KINDS (nothing dispatches on kind). (2) Replaced the hand-rolled single-file SVG renderer with a Vite + React + React Flow (@xyflow/react v12) + elkjs app under `viz/` (user picked this stack after a build-vs-no-build options review). `build_all.py` no longer bakes `const DATA` into HTML — it writes `graphs.json` to `traces/` and `viz/public/`, which the app fetches. Custom `TaskNode` keeps the card UI (kind, label, exact FLOPs, proportional bar, colour by kind); edges keep rejected/fan-in/handoff styling. Adversarial review (general-purpose agent) found no BLOCKER/MAJOR; applied its 2 cheap MINORs (fetch abort guard, edgeStyle null guard). Roadblock: a 68-node chain under `fitView` collapsed to an unreadable ribbon → enabled elk `MULTI_EDGE` wrapping at aspectRatio 1.7 + capped `maxZoom` 1.4. Verified headlessly with playwright/chromium against the LIVE surge URL — all four tabs render with correct node/edge counts + FLOPs totals, zero console errors. dist/ + node_modules/ gitignored. Live: https://taskgraph-dss.surge.sh
