tap-protocol — protocol viewer

Animates the 4-server inference verification flow. Default sample is a real 9-request capture from a Qwen3-1.7B run on H100 (vast.ai instance 39076914); also ships synthetic samples covering the alarm path. Schema is documented below; the viewer can also connect live to a running Tap via ?mode=live&url=….

Event schema 
Each event: {ts, type, id, ...payload}

ts:    seconds since stream start
id:    monotonic request id (matches SignedEnvelope.data.id)
type:
  request_sent             client → gateway (plain prompt)
  gateway_signed           gateway computed HMAC; payload: signature_prefix
  tap_received             gateway → tap (signed envelope)
  tap_relayed_request      tap → host (relay)
  host_completed           host returned; payload: output_preview,
                           output_sha256, tokens
  tap_relayed_response     host → tap → gateway (signed envelope)
  client_received          gateway → client (plain output)
  tap_verify_started       tap → recomp (async copy)
  recomp_verified          recomp returned; payload: is_verified,
                           expected_sha256, actual_sha256, reason?

Mode toggle:
  ?mode=live&url=...  poll Tap's /events SSE endpoint
  (default)           replay embedded sample or uploaded file
